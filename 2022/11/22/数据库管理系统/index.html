数据库三天速成

sql 语句分类：
  • 表的结构
    • 行（被称为数据data）
    • 列（被称为字段column）
      • 每一个字段应该包括：字段名、数据类型、数据相关约束
  • 通用sql语句如下：
    • DQL（数据查询语句）
      • 凡是查询语句select就是查询语句
    • DML（数据操作语句）
      • insert delect update —对表中的数据进行增删改查
    • DDL（数据定义语句）
      • create drop alter —对表结构进行增删改查
    • TCL（事务控制语言 T -> transaction）
      • commit 提交事务 、rollback 回滚事务 、save point 保存点
    • DCL（数据控制语言）
      • grant 授权 、revoke 撤销权限

数据排序：

  • select 字段1，字段2，字段3…. from table order by 字段/或者是数字（代表的是字段的列） ；

    • 默认是为升序 asc

    • 不建议用数字不健壮

    • 降序为 desc

    • select ename ,sal from emp order by sal asc,ename desc;
// 就是说上面的先按照第一个排序为大前提，然后再第二个为次前提
// 越靠前的越占主导作用

    • 以上五个分组函数都是对“某一组”数据进行操作

      • select 函数（字段）from 字段；

    • 分组函数也可以联合起来一起用

分组查询 group by：

  • group by 按照某个字段或者是某个字段进行分组

    • mysql> select max(sal),job from emp group by job;
+----------+-----------+
| max(sal) | job       |
+----------+-----------+

|  1300.00 | CLERK     |
|  2975.00 | MANAGER   |
|  5000.00 | PRESIDENT |
|  1600.00 | SALESMAN  |
+----------+-----------+

    • 注意：

      • ```sql
        select ename ，max(sal),job from emp group by job;

  • having和where的选择

    • having是group by 的搭档，对于已经的分组在进行过滤
    • where进行过滤的效率的高，having的效率低一点，因为在having之前已经进行了分组，到了having又对分的组进行过滤，使效率低下
去除重复记录：
  • select distinct job from emp;  //只能出现在所有字段的最前面

select distinct epmtno,job from emp;  //distinct后面的字段的联合去重

select count(distinct job) from emp;  //去除重复的之后再进行统计

  • 非等值连接（内连接）

    • 最大特点就是：连接条件中的等量关系是非等量关系。

    • ```sql
      //找出员工的薪资等级
      selec
      e.ename,e.sal,s.grade
      from
      emp e
      join
      salgrade s
      on
      e.sal between s.losal and s.hisal;

  • from后面跟上一个查询语句形成的表

    • ```sql
      //找出每个部门的平均工资的等级
      //将新形成的表跟在from后面，再进行内连接
      //要有一个临时表的思想
      select
      t.*,s.grade
      from
      (select deptno,avg(sal) avgsal from emp group by deptno) t
      join
      salgrade s
      on
      t.avgsal between s.losal and s.hisal;

  • 1.要求多条查询语句的查询列数是一致的！

  • 2.要求多条查询语句的查询的每一列的类型和顺序最好一致

  • 3.union关键字默认去重，如果使用union all 可以包含重复项

limit （※）：

  • limit是mysql中特有的，很重要，其他数据库没有不通用

  • 分页查询全靠它了

  • limit是去、取结果集中的部分数据

  • 语法机制

    • ```sql
      Limit startindex, length
      // startindex 从该下标之后开始
      //length 步长
      //以上的意思就是：从第几个之后开始，输出多少个
      //limit方法中不能参与运算
      //limit中：This version of MySQL doesn’t yet support ‘LIMIT & IN/ALL/ANY/SOME subquery’—->淦

  • mysql中的字段类型：

    • int 整数型
    • bigint 长整数型（就是Java中的long）
    • float 浮点型
    • char 定长字符串
    • varchar 可变长度字符
    • date 日期类型（对应Java中的Java.sql.date）
    • BLOB 二进制大对象（存储图片，视频等流媒体信息）—–>普通的insert是不行的，得用I/O流，现在的都是在数据库中存储的路径，但是图片都是放在其他地方如硬盘等
    • CLOB 字符大对象（存储比较大的文本，比如4GB的字符串）—–>和上面的那个产不多

  • 一般建议的表都是：t_ 或者tbl_ 开始

  • insert语句插入数据：

    • insert into 表名（字段名1，字段名2，….） values(值1，值2，….)

    • 就是只要前后对的上数据类型和个数就行了

    • 只要执行成功，表中就会增加一条记录

    • 可以通过default来设置默认值

    • 字段可以省略不写，但是values后面一定要要个匹配哦

    • //一次可以插入多条记录
insert into 
    表名（字段名1，字段名2，....）

    (值1，值2，....),(值1，值2，....),....

    • ```sql
      insert into 新表名 select 语句

  • 表的删除：

    • delete是把目录撕了，truncate是把书的内容撕下来烧了，drop是把书烧了。
    • 用delete删除大表的效率是比较低的，因为删除的时候其中的表格是没有被删除的，在进行回滚的时候是可以进行恢复的
    • 用truncate删除之后是没有办法恢复了的，一定要问清楚才行，是不可以回滚的

约束：（constraint）

即使保证数据的合法性

  • 非空约束（not null）:约束的字段不能为空

    • 没有表级的约束

  • 唯一约束（unique）:约束的字段不能重复

    • 修饰的字段，就是不能重复，但是可以为null（null为空就是不算为重复的）

    • create table t_user(
id varchar(255),
username varchar(255) unique ,
unique (字段1，字段2)
);
//表级约束就是说联合约束--》unique（字段1，字段2，。。。）
//列级约束就是直接添加在字段后面的约束--》字段1  数据类型  unique
//也可以是直接在表级上面写一个就直接是列级了

    • 主键术语？

      • 主键约束
      • 主键值
      • 主键字段

    • 主键的作用？

      • 表的设计三范式中有要求，第一范式要求每一张表都要求有主键
      • 每行记录的唯一标识

    • 划分方式：

      • 数量
        • 单一主键 (推荐使用)
        • 复合主键（多个字段联合起来添加的，不建议使用，违背三范式）
      • 性质
        • 自然主键：主键值最好是一个和业务不挂钩的自然数就ok了
        • 业务主键：与业务有关系。如银行卡号，身份证号（不过这是不推荐使用的），因为一旦业务发生改变的时候，主键值也需要发生改变。

    • 注意：一张表只能拥有一个主键

    • mysql提供主键值自增：（important）—-》主键自动维护 auto_increment

      • 就是说主键这个东西，不用用户进行自己设置，不然用户会疯掉的

      • drop table if  exist t_user;
create table t_user(
    id int primary key auto_increment,  
    username varchar(255)
);
insert into t_user(username) values('a');
insert into t_user(username) values('b');
insert into t_user(username) values('c');
//执行结果就是说，主键id字段的值自动增加，并且是从1开始以1递增

      • 在删除表时只能先删掉子表再删掉副表，因为子表还在用父表的值

      • 外键一般情况下，references的不一定是主键，但是至少具有unique约束（因为被引用的数据必须具有唯一性）

      • 外键值：可以是NUll

  • 检查约束（check）：该约束就是在Oracle当中才有，在MySQL当中没有该约束

存储引擎（了解）—面试会问到：

  • MySQL特有的这个名字’存储引擎’：表的存储方式

  • 不同的存储引擎导致不同的在底层的存储方式不同，每个都有自己的优点和缺点

  • InnoDB就是默认的存储引擎

  • ```sql
    CREATE TABLE emp (
    EMPNO int(4) NOT NULL,
    ENAME varchar(10) DEFAULT NULL,
    JOB varchar(9) DEFAULT NULL,
    MGR int(4) DEFAULT NULL,
    HIREDATE date DEFAULT NULL,
    SAL double(7,2) DEFAULT NULL,
    COMM double(7,2) DEFAULT NULL,
    DEPTNO int(2) DEFAULT NULL,
    PRIMARY KEY (EMPNO)
    ) ENGINE=InnoDB DEFAULT CHARSET=utf8
    // ENGINE=InnoDB 这就是一个存储引擎
